⭐ The engine

Recursive self-improvement

This library is not a wiki that rots. It is a loop that runs. Every task executed by the team — human or AI — passes through five gates, and the documentation comes out sharper than it went in.

Do → Document (skill.md) → QA (Definition of Done) → Example (Meta-Article) → Improve (SOP Update) → ↺

Every task we perform is captured as a skill.md — a step-by-step SOP with a QA gate (the Definition of Done) and a worked example (the meta-article). The next person to run the task, human or AI agent, starts from the best documented run instead of from scratch.

Each pass through the loop sharpens the procedure, tightens the checklist, and adds proof. Nothing is ever done twice from zero. The system compounds.

  • ✓One standard, zero drift — all 239 skills follow the same definitive-article template, section for section.
  • ✓QA gate on every task — a Definition of Done that a stranger (or an agent) can verify without asking.
  • ✓Example slot in every SOP — each skill links the meta-article that proves the process works.
  • ✓Self-updating — every run ends by improving the very document it ran from.
