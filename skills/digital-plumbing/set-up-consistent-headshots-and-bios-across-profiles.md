# Set Up Consistent Headshots and Bios Across Profiles

**Use this when** profiles show different photos, ages, names, or bios — the inconsistency that keeps Google from connecting the dots into one entity (and one Knowledge Panel).

## Inputs
- One current, professional, real headshot (no stock, no AI portrait, no decade-old photo)
- Canonical bio in two lengths: short (fits Instagram/X-style limits) and long (LinkedIn/YouTube/About page), written in first person
- Logins for every profile: Facebook, Instagram, YouTube, LinkedIn, GBP, plus the website About page

## Steps
1. Lock the canon: with the owner, pick the one headshot and finalize short + long bios. Same name spelling everywhere — the exact form used on the website and schema.
2. Store the canon (image files + bio text) in the Content Library so every future profile pulls from the same source.
3. Apply to each surface in one pass: Facebook page, Instagram, YouTube channel, LinkedIn, Google Business Profile, the website About/hero, and the owner's email avatar.
4. Use the long bio where space allows, the short bio where it doesn't — but never a different story. Each bio states who they help, what they do, and one piece of proof.
5. Verify visually: open all profiles side by side; the same face and the same first line should greet you on every one.
6. Re-run this skill whenever the headshot or positioning changes — partial updates recreate the inconsistency this task exists to kill.

## Definition of done (QA checklist)
- [ ] Identical headshot live on Facebook, Instagram, YouTube, LinkedIn, GBP, and the website
- [ ] Bio copy consistent across all profiles (short/long variants of one canon, first person)
- [ ] Name spelling identical everywhere, matching website and schema
- [ ] Canon files stored in the Content Library
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Run on a persistent agent (Fable 5)
A persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) applies the canon in one sweep, then opens every profile side by side to confirm the same face and first line greet it on each — looping until the full Definition of done passes, not 90% of profiles.
Memory is the point here: the canon headshot and bios persist across runs, so when positioning changes the agent re-runs the whole sweep instead of patching one profile, self-verifying against that checklist and logging a meta-article example each run so the library compounds.
