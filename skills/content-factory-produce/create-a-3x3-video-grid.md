# Create a 3×3 video grid

**Use this when** the WHY video exists and the brand needs systematic topic coverage that can later be sequenced cold → warm → conversion in Dollar a Day.

## Inputs
- The completed 3-minute WHY video (anchor asset — see `create-a-3-minute-why-video`)
- Three core topics chosen from the Topic Wheel HOW ring (your methods, what you want to be known for)
- A phone and the on-camera person; a half-day recording block
- Access to the Content Library Drive folder

## Steps
1. Pick the 3 core topics from the Topic Wheel HOW ring. These should match the SEO Tree trunks the website is built on — content and site architecture mirror each other.
2. For each topic, plan 3 videos mapped to the three funnel levels (Audience, Engagement, Conversion — the funnel triangle): one awareness video (the WHY behind the topic, a story), one engagement video (the HOW — method, framework, walkthrough), one conversion video (the WHAT — proof, results, offer, customer story).
3. Draw the grid: 3 topics × 3 funnel levels = 9 cells. Write one real customer question or story prompt in each cell before recording anything.
4. Record all nine as one-minute-video style takes: phone, unscripted, one take each (see `record-one-minute-videos`). Batch them in one session.
5. Name files `YYYY-MM-DD-topic-funnelstage` and upload all nine to the Content Library Drive folder.
6. Log the grid in the Content Library tracker as a unit — note each video's topic, funnel level, and grid position so Promote can sequence them cold → warm → conversion.
7. Queue the nine for the Process stage; each becomes an article, clips, and ad creative.

## Definition of done (QA checklist)
- [ ] Exactly 9 videos: 3 topics × 3 funnel levels, every cell filled
- [ ] Topics come from the Topic Wheel HOW ring and match the site's SEO Tree trunks
- [ ] Each video is phone-shot, unscripted, ~60 seconds, one idea
- [ ] Grid positions logged so Dollar a Day sequencing (cold → warm → conversion) can use them
- [ ] All nine uploaded to the Content Library and queued for Process
- [ ] Linked back to the definitive article and relevant siblings

## Run on a persistent agent (Fable 5)

Nine cells, all filled: a persistent agent (Claude Fable 5, or comparable long-horizon OpenAI/Google models) builds the grid from the Topic Wheel HOW ring, writes a real question into every cell before recording starts, then loops the post-session check until 9/9 are uploaded, positioned, and logged — eight of nine is not done. Memory holds grid state across sessions, so a half-finished grid resumes instead of restarting. Log a meta-article example when the grid completes.
See `boil-the-ocean.md` for the full operating principles.
