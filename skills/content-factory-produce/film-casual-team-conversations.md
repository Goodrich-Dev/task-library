# Film casual team conversations

**Use this when** the team is genuinely working through a problem — a client strategy debrief, a post-mortem, a "how do we fix this" huddle — and the conversation is worth overhearing.

## Inputs
- A real working conversation about to happen (stand-up, strategy session, client debrief) — never staged
- A phone propped where it can see and hear the participants
- Consent from everyone on camera, and agreement on what client details stay private
- Access to the Content Library Google Drive folder

## Steps
1. Get quick consent from everyone in the room ("we're recording this for content — anything off-limits?") and note any client names or numbers to cut later.
2. Prop the phone at eye level with the whole group in frame; landscape for discussion footage. Start recording before the conversation warms up.
3. Do not stage, script, or re-shoot. The value is real people solving a real problem — disagreement and thinking-out-loud included.
4. Let it run. Do not stop-start for "good takes"; Process will find the moments.
5. While memory is fresh, jot timestamps of the strongest 2–3 moments (a sharp insight, a clear explanation, a decision being made).
6. Name the file `YYYY-MM-DD-topic-team-convo` and upload to the Content Library Drive folder.
7. Log in the Content Library tracker with the timestamp notes and any redaction flags, status "Raw — ready for Process."

## Definition of done (QA checklist)
- [ ] Conversation is real work, not performed for the camera
- [ ] Everyone visible gave consent; off-limits client details flagged
- [ ] Audio is intelligible for all speakers (transcription-grade)
- [ ] Strongest moments timestamped in the tracker notes
- [ ] File uploaded to the Content Library and logged for the Process stage
- [ ] Linked back to the definitive article and relevant siblings

## Run on a persistent agent (Fable 5)

The team talks; a persistent agent (Claude Fable 5, or a comparable long-horizon OpenAI/Google model) does the follow-through — looping post-recording until consent notes, timestamped highlights, redaction flags, and the tracker row all pass the Definition of done, never settling for "uploaded, mostly logged." Memory holds which conversations were captured and what was flagged off-limits, so nothing private leaks into Process later. Each run ends with a meta-article example logged.
