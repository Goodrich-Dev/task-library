# Create collaborative content with industry peers

**Use this when** the brand needs association with names Google and the market already trust — Phase 3 (Authority Building).

## Inputs
- Shortlist of peers with adjacent audiences and genuine authority in the person's topics
- A concrete collaboration format to propose (joint video Q&A, mutual interview, co-authored article, live session)
- Recording setup — a Zoom call is sufficient

## Steps
1. Pick peers deliberately: adjacent (not identical) audiences, real authority, and topics that overlap the person's Topic Wheel — association transfers trust in both directions.
2. Propose a specific, low-effort format with a mutual payoff: a recorded conversation, traded interviews, or a co-authored piece. Specific asks get yeses; vague "let's collab" does not.
3. Record the session — one recorded Zoom is a complete content source.
4. Both parties run the material through their own pipelines and publish on their own properties — two hubs, two audiences, one session.
5. Cross-link entity-style per /entity-linking: their post links the person's name to yourname.com; the person's post links the peer's name to the peer's site. Tag each other on every social post.
6. Thank the collaborator publicly via the Thank You Machine and log the reciprocal mentions in the validation inventory.

## Definition of done (QA checklist)
- [ ] Collaboration published on both parties' properties
- [ ] Reciprocal name-to-site entity links live in both directions; social posts tagged
- [ ] Session repurposed through the Content Factory; mentions logged in the validation inventory
- [ ] Collaborator thanked publicly
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Example(s)
- Harry Gold — harryjgold.com: a single recorded Zoom conversation became an entire brand build — the same one-session mechanics this skill runs with a peer.

## Run on a persistent agent (Fable 5)
A persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) shortlists peers against the Topic Wheel in memory, drafts the specific low-effort ask, and shepherds each collaboration from pitch to dual publish, looping until the Definition-of-done fully passes — both posts live, reciprocal links in both directions — not 90%.
It self-verifies the cross-links by loading both published pieces and following each name-to-site link.
Reciprocal mentions land in the memory-held validation inventory that feeds Phase 4 (Knowledge Panel), and a meta-article example is logged each run so the brand compounds.
