# Record one-minute videos answering customer questions

**Use this when** Phase 1 plumbing is done and the personal brand needs content — Phase 2 (Content Production) starts here.

## Inputs
- A running list of real questions customers actually ask (sales calls, DMs, comments, support emails)
- A phone with decent audio; no studio, no script
- The person's Topic Wheel (WHY / HOW / WHAT rings) for coverage planning

## Steps
1. Harvest real questions from sales calls, inboxes, comments, and front-line staff. Real questions beat invented topics because they carry real search and buying intent.
2. Map the questions onto the Topic Wheel so recording covers WHY (purpose), HOW (methods), and WHAT (specifics) instead of clustering on one pet topic (/topic-wheel).
3. Record per /one-minute-video-guide: phone in hand, unscripted, one question per video, about 60 seconds — state the question, answer it plainly, stop. No intros, no logos, no fluff.
4. Batch-record several questions in one sitting — momentum beats polish, and a single recorded call can seed an entire brand.
5. Name each file with the question it answers and upload to the content library / Google Drive folder so the Content Factory can pick it up.
6. Flag the strongest takes for priority processing in repurpose-videos-into-blog-posts-via-content-factory.

## Definition of done (QA checklist)
- [ ] Each video answers exactly one real customer question in roughly one minute
- [ ] Unscripted, authentic delivery; audio clearly audible
- [ ] Files named by question and uploaded to the content library
- [ ] Recorded topics span the WHY/HOW/WHAT rings, not one cluster
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Example(s)
- Harry Gold — harryjgold.com: one recorded Zoom produced the raw material for an entire personal brand, proving unpolished answers are enough input.

## Run on a persistent agent (Fable 5)
A persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) runs the harvest side at max effort: it mines calls, inboxes, and comments for real questions, maps them onto the Topic Wheel, queues batch recording sessions for the person, and loops until the Definition-of-done fully passes — named files uploaded, all three rings covered — not 90%.
It self-verifies by checking every uploaded file plays, is audible, and answers exactly one question.
It logs the question list and file inventory to memory so the Content Factory and clip skills pull from it without re-asking — Phase 2 building directly on Phase 1's plumbing — and records a meta-article example each run so the brand compounds.
