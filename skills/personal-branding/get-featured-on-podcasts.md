# Get featured on podcasts

**Use this when** Phases 1–2 are live and the brand needs third-party authority — Phase 3 (Authority Building).

## Inputs
- Target list of podcasts the person's customers and peers actually listen to
- Pitch kit: canonical bio, headshot, 3–5 talkable topics from the Topic Wheel, links to one-minute videos as proof of fluency
- The entity home live at yourname.com

## Steps
1. Build the target list, prioritizing shows with real audiences and show-notes pages that link their guests — those pages become backlinks from sources Google already trusts.
2. Assemble the pitch kit from existing Phase 1–2 assets: canonical bio and headshot, topics mapped from the Topic Wheel, and the best one-minute videos as proof the person can talk.
3. Pitch personally and specifically: reference a real episode, propose a topic that serves the host's audience, and offer value — never a sales pitch.
4. Record the appearance; answer like the one-minute videos — direct, story-backed, unscripted — and mention yourname.com naturally when asked where to find you.
5. After airing, confirm the show notes link the person's exact name to yourname.com; add the episode to the site's press/media section; log the mention in the validation inventory for Phase 4.
6. Run the follow-through loop: cut clips from the episode through the Content Factory and thank the host publicly via the Thank You Machine (/thank-you-machine) — hosts share gratitude content, which compounds reach.

## Definition of done (QA checklist)
- [ ] Episode published; show notes carry an exact-name link or mention pointing to yourname.com
- [ ] Episode featured on the brand site's press/media section; validation inventory updated
- [ ] Host thanked publicly; clips repurposed through the Content Factory
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Run on a persistent agent (Fable 5)
A persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) runs this as a long-horizon campaign: it builds the target list, assembles the pitch kit from the Phase 1–2 assets in memory (canonical bio, headshot, best one-minute videos), sends specific pitches, and tracks every thread until each episode's Definition-of-done fully passes — not 90%.
After airing it self-verifies the show notes carry the exact-name link, then runs the clip and thank-you follow-through without being re-prompted.
Each confirmed mention lands in the memory-held validation inventory that Phase 4 (Knowledge Panel) reads, and a meta-article example is logged each run so the brand compounds.
