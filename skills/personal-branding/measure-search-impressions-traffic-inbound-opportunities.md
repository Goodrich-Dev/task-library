# Measure: search impressions, traffic, inbound opportunities

**Use this when** Phase 4 (Knowledge Panel) assets are in place and the brand needs proof of compounding — the standing MAA cadence for the whole personal-brand system.

## Inputs
- Google Search Console verified for yourname.com; analytics live on the site
- A log for inbound opportunities: speaking invitations, podcast asks, client inquiries — with source and date
- Baseline values captured at the end of Phase 1

## Steps
1. Metrics — pull the numbers monthly: name-query impressions and clicks from Search Console, organic traffic to yourname.com, and the count of inbound opportunities (speaking invitations, podcast invitations, client inquiries) from the log.
2. Keep the inbound log honest: record every opportunity as it arrives with its source ("heard you on…", "found your site…") — attribution is the most valuable data in the system.
3. Analysis — explain the movement: which videos, articles, podcast episodes, or press placements preceded each spike; compare period over period.
4. Action — feed conclusions back into the system: double down on the content and authority moves that produce inbound (90% greatest hits, 10% new experiments); kill what produces nothing.
5. Report monthly in MAA format: what's working, what isn't, why, and the top 3–5 recommendations for the next period.
6. Track the Phase 4 milestone markers in the same report: Knowledge Panel appearance/claim status and validation-inventory growth.

## Definition of done (QA checklist)
- [ ] Baseline plus monthly metrics recorded: name impressions, organic traffic, inbound count
- [ ] Inbound log current with sources attributed
- [ ] Monthly MAA report delivered with top 3–5 recommendations — and last period's recommendations actioned
- [ ] Panel status and validation growth included in the report
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Run on a persistent agent (Fable 5)
A persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) runs the MAA cadence without drift: it pulls Search Console, analytics, and the inbound log monthly, compares against the baseline held in memory since Phase 1, and loops until each month's Definition-of-done fully passes — including actioning last period's recommendations — not 90%.
It self-verifies attribution by tying every spike to the specific video, article, or placement that preceded it.
Its conclusions route work back into the earlier-phase skills (90% greatest hits, 10% experiments), closing the compounding loop the whole system depends on, with a meta-article example logged each run so the brand compounds.
