# Generate backlinks and entity mentions

**Use this when** the entity home and content engine are live and the brand needs external proof — runs continuously through Phase 3 (Authority Building).

## Inputs
- Baseline inventory of current backlinks and name mentions pointing at yourname.com
- Target list of trusted sources: industry publications, podcast show notes, event speaker pages, association directories, partner and client sites
- The canonical name string (mentions must match it exactly)

## Steps
1. Record the baseline — referring domains and name mentions for yourname.com — so the MAA loop has a starting line.
2. Prioritize sources Google already trusts: industry publications, established podcasts, event sites, associations the person belongs to, and partners or clients who can credit work publicly.
3. Earn placements, never buy them: supply expert quotes, contribute articles, and harvest the links generated by the sibling skills (podcasts, speaking, collaborations, press).
4. For every placement, request the exact canonical name as the mention, linked to yourname.com where editorially possible — name consistency is what turns a link into an entity signal.
5. Where coverage needs warming, run Dollar a Day at journalists and media per /dad to put the person's best content in front of them before the pitch.
6. Log every new mention and link in the validation inventory (the Phase 4 input), and review monthly via MAA: referring domains, mention count, and name-search impressions.

## Definition of done (QA checklist)
- [ ] New backlinks or mentions earned this cycle from trusted sources — no paid link schemes
- [ ] Every mention uses the exact canonical name; links point to yourname.com
- [ ] Validation inventory and baseline updated; monthly MAA review held
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Run on a persistent agent (Fable 5)
This task runs continuously, so a persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) fits it natively: it holds the baseline in memory, harvests links produced by the sibling Authority skills, pitches new placements, and loops each monthly cycle until the Definition-of-done fully passes — not 90%.
It self-verifies every placement — exact canonical name, live link, genuinely trusted source — and rejects anything resembling a paid link scheme.
The validation inventory and MAA baseline it maintains in memory are exactly what the Knowledge Panel phase reads next, and a meta-article example is logged each run (this file still needs its first) so the brand compounds.
