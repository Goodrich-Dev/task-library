# Get mentioned in publications

**Use this when** the brand has authority assets (podcasts, talks, results) and needs recognized press — Phase 3 (Authority Building). This task has no definitive article yet; flag the gap on every run.

## Inputs
- Target list of recognized publications the person's buyers — and Google — trust (industry trades, business press, local media)
- Press kit: canonical bio, headshot, evidenced credentials, links to prior features and best content
- Newsworthy material: data, results, a contrarian position, or a timely expert take

## Steps
1. Build the target list: publications that cover the person's industry, ranked by recognition and editorial standards — recognized outlets are exactly the validation Google weighs.
2. Assemble the press kit from existing assets: canonical bio and headshot, credentials with evidence, and prior podcast or speaking features as social proof.
3. Create reasons to be covered: original data or client results, a defensible contrarian take pulled from the one-minute videos, or news (a talk, a launch, a milestone).
4. Pitch specific story angles to specific journalists, and answer journalist source requests fast with quotable, evidence-backed responses.
5. Warm targets where useful by running Dollar a Day at journalists and media per /dad before and after the pitch.
6. When published: verify the exact canonical name is used, request a link to yourname.com where editorially possible, add the piece to the brand site's press section, and log it in the validation inventory for Phase 4.
7. Flag the missing hub: this task's definitive article is a GAP — note it on the Gaps & Tasks to Create list so the hub gets written.

## Definition of done (QA checklist)
- [ ] At least one new mention in a recognized publication this cycle
- [ ] Mention uses the exact canonical name; link to yourname.com requested (secured where editorially possible)
- [ ] Press section and validation inventory updated
- [ ] Missing definitive article flagged on the gaps list
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Run on a persistent agent (Fable 5)
A persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) runs press as a standing campaign: it maintains the target list, assembles the press kit from the Phase 1–3 assets in memory, pitches specific angles, answers source requests fast, and loops until each cycle's Definition-of-done fully passes — not 90%.
It self-verifies every published mention: exact canonical name used, link requested or secured, press section and validation inventory updated.
It also re-flags the missing definitive article on the gaps list every run until the hub is written, and logs a meta-article example each run so the brand compounds.
