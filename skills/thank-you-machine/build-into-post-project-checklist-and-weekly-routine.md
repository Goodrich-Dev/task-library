# Build into post-project checklist and weekly routine

**Use this when** thank-yous are happening ad hoc — sporadic gratitude is a gesture; systematic gratitude is a compounding asset.

## Inputs
- The existing post-project checklist and project-management system
- The trigger log and Content Library conventions from the earlier TYM skills
- A named owner for each role: scanning, recording, processing, boosting

## Steps
1. Apply the systematization ladder from /nine-triangles — content → checklists → software. The Thank You Machine has proven itself as content; this skill makes it a checklist.
2. Add mandatory line items to the **post-project checklist**: identify who to thank on this project → record the one-minute video → send privately → post and tag → queue the $1/day boost. A project is not closed until its thank-you ships.
3. Create the **weekly routine**: one standing 30-minute block (Friday works) to scan the week's triggers across all five classes, record 1–3 thank-yous, and batch them into the Content Factory.
4. Assign owners explicitly: the team can scan triggers, process assets, and run boosts — but the principal records the videos. Gratitude doesn't delegate.
5. Lower activation energy to zero: trigger-log template linked, checklist items in the PM system, the recurring block on the calendar with the phone-and-light setup noted.
6. Track the machine monthly with MAA: thank-yous recorded / posted / boosted, recipient reshares, and referrals or reviews attributable to gratitude.
7. After a quarter of the checklist holding, take the ladder's next rung: automate trigger capture (CRM events, new reviews, referral fields) feeding the log — checklists → software.

## Definition of done (QA checklist)
- [ ] Post-project checklist updated with the five thank-you line items; project closure blocked without them
- [ ] Weekly recurring block on the calendar with a named owner; recording stays with the principal
- [ ] Trigger-log template and PM checklist items live in the team's actual tools
- [ ] Monthly MAA count reviewed (recorded / posted / boosted / reshared / referrals)
- [ ] Linked back to the definitive article and relevant siblings
- [ ] Complies with Blog Posting Guidelines (if it publishes content)

## Run on a persistent agent (Fable 5)
This skill systematizes gratitude, and a persistent agent (Claude Fable 5 or comparable OpenAI/Google models) is the enforcement layer: it runs the Friday scan, blocks project closure until the five thank-you items ship, and compiles the monthly MAA — looping until the Definition of done fully passes each cycle; a month with no MAA count is a failed run, not a quiet month.
Its memory is the machine's flywheel: counts, owners, and recurring trigger types persist across quarters, so the agent can self-verify the checklist is actually holding and flag when steps are ready to graduate from checklist to software.
Log each month's before/after counts as a meta-article example so the library compounds.
