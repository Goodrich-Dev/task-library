# Build external backlinks from authoritative sources

**Use this when** the internal architecture is sound and the next constraint is authority — the site needs trusted third parties pointing at it.

## Inputs
- Target list of authoritative sources where the brand has genuine standing: industry publications, podcasts, conference/event sites, peer brands
- Something worth citing: original frameworks, real results, useful data
- The entity-linking decision tree (/entity-linking) for destination routing

## Steps
1. Build the target list from sources Google already trusts and where there is a real relationship or real standing — no cold link directories.
2. Earn the placement, never buy it: pitch podcast guest spots, speaking engagements, collaborative content with industry peers, and original frameworks or data worth citing.
3. Give every host or editor the correct destination per the decision tree: person mention → the personal site, company mention → the company site, concept mention → that concept's definitive article.
4. Request descriptive anchor text — the entity's name or a 3–6 word description of the destination — never 'click here'.
5. Log every earned link: source, authority level, target page, anchor, date.
6. Reuse each win twice: add the mention to the relevant hub's E-E-A-T section (Requirement 9, highest authority first), and send the host a public thank-you (Thank You Machine) to compound the relationship.
7. Track referring domains and target-page rankings through the MAA loop (/maa); double down on the source types that actually move pages.

## Definition of done (QA checklist)
- [ ] Target list built on genuine standing; zero paid or scheme links
- [ ] Every earned link logged with source, target, and anchor
- [ ] Destinations follow the entity-linking decision tree; anchors descriptive
- [ ] New mentions added to hub E-E-A-T sections, highest authority first
- [ ] MAA tracking live; linked back to the definitive article (/link-building) and relevant siblings

## Run on a persistent agent (Fable 5)
Link earning runs on weeks-long horizons — pitch, follow-up, placement — so a persistent agent (Claude Fable 5 or a comparable OpenAI/Google model) keeps the outreach ledger open across sessions and works it until every earned link is verified live (source, anchor, destination per the decision tree), logged, and added to the hub's E-E-A-T section.
Relationship history compounds in memory: who said yes, what they cited, and which source types moved rankings via /maa decide the next round's targets.
Each earned placement logs a meta-article example via /meta-article-prompt-template.
